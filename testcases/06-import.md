# TC-06 · Импорт транзакций (CSV)

**Предусловие:** пользователь авторизован, открыта страница `/import`.

---

## TC-06-01 · Загрузка корректного CSV файла

**Предусловие:**
- Подготовлен файл `test.csv` с корректными строками:
  ```
  date,title,amount,category,tag
  2026-06-01,Кофе,350,expense,еда
  2026-06-02,Зарплата,50000,income,работа
  ```

**Шаги:**
1. На странице `/import` перетащить или выбрать файл `test.csv`

**Ожидаемый результат:**
- Файл загружен
- Отображается таблица модерации со строками из файла
- Строки подсвечены зелёным (без ошибок)

**Playwright Hint:**
```typescript
const fileInput = page.locator('input[type="file"]');
await fileInput.setInputFiles('tests/fixtures/test.csv');
await expect(page.getByText('Кофе')).toBeVisible();
```

## TC-06-02 · CSV с неизвестными метками

**Предусловие:**
- CSV с меткой, не существующей в системе: `tag=несуществующая`

**Шаги:**
1. Загрузить файл

**Ожидаемый результат:**
- Строка с ошибкой подсвечена красной границей в поле «Метка»
- Счётчик ошибок вверху: «У вас N транзакций с ошибками — наведите на выделенное поле чтобы увидеть подробности»
- При наведении на поле метки — tooltip с текстом ошибки

---

## TC-06-03 · CSV с некорректной категорией (enum error)

**Предусловие:**
- CSV содержит `category=покупка` (вместо `expense`/`income`)

**Шаги:**
1. Загрузить файл

**Ожидаемый результат:**
- Поле категории выделено красным
- В tooltip — «значение "покупка" недопустимо, ожидается expense или income»
- Транзакция НЕ создаётся в БД (нет ошибки 500)

---

## TC-06-04 · CSV с некорректной суммой

**Предусловие:**
- CSV содержит `amount=тысяча`

**Шаги:**
1. Загрузить файл

**Ожидаемый результат:**
- Поле суммы выделено красным
- В tooltip — ошибка с указанием некорректного значения

---

## TC-06-05 · Исправление ошибки в модерации

**Шаги:**
1. Загрузить CSV с ошибкой (например, пустая метка)
2. В таблице модерации выбрать правильную метку в соответствующем поле
3. Ошибка исчезает

**Ожидаемый результат:**
- Поле больше не выделено красным
- Счётчик ошибок уменьшился

---

## TC-06-06 · Принятие всех строк (Accept All)

**Шаги:**
1. Загрузить корректный CSV
2. Нажать «Принять все» (AcceptBatch)

**Ожидаемый результат:**
- Все строки без ошибок приняты
- Создаются транзакции
- Страница возвращается в состояние «нет активного импорта»

---

## TC-06-07 · Принятие отдельной строки

**Шаги:**
1. Загрузить CSV
2. Нажать «✓» (accept) на конкретной строке

**Ожидаемый результат:**
- Строка исчезает из таблицы
- Создана одна транзакция

---

## TC-06-08 · Закрытие активного батча

**Шаги:**
1. Загрузить CSV (есть активный батч)
2. Нажать «Закрыть» / «Отменить импорт»

**Ожидаемый результат:**
- Активный батч закрыт
- Таблица модерации исчезает
- Можно загрузить новый файл

---

## TC-06-09 · Принятие строк с ошибками заблокировано

**Шаги:**
1. Загрузить CSV с ошибками
2. Попытаться принять строку с ошибкой

**Ожидаемый результат:**
- Кнопка «Принять» неактивна для строк с ошибками
- Или появляется предупреждение
