### Requirement: Иерархия тегов и подтегов

Система SHALL поддерживать пользовательские теги с произвольной вложенностью подтегов (дерево: тег → подтег → …).

#### Scenario: Создание корневого тега
- **WHEN** пользователь создаёт тег без родителя с уникальным названием
- **THEN** тег сохраняется с `parent_id = null` и отображается в дереве тегов

#### Scenario: Создание подтега
- **WHEN** пользователь создаёт подтег, указав родительский тег
- **THEN** подтег сохраняется с `parent_id` родителя и отображается вложенным в дереве

### Requirement: Страница управления тегами

Приложение SHALL предоставлять отдельную страницу управления тегами с CRUD-операциями.

#### Scenario: Редактирование названия тега
- **WHEN** пользователь изменяет название существующего тега и сохраняет
- **THEN** новое название отображается в дереве и доступно в селекторах транзакций

#### Scenario: Удаление неиспользуемого тега
- **WHEN** пользователь удаляет тег, не привязанный ни к одной записи
- **THEN** тег удаляется из дерева без предупреждения об использовании

#### Scenario: Удаление используемого тега
- **WHEN** пользователь пытается удалить тег, используемый в транзакциях, лимитах или платежах
- **THEN** система показывает предупреждение о количестве связанных записей и запрашивает подтверждение

### Requirement: Inline-создание тегов в форме транзакции

При добавлении транзакции пользователь SHALL иметь возможность создать новый тег или подтег прямо в форме, без перехода на страницу управления.

#### Scenario: Создание тега из формы транзакции
- **WHEN** пользователь вводит название нового тега в селекторе тега формы транзакции
- **THEN** тег создаётся и сразу выбирается для текущей транзакции

#### Scenario: Создание подтега из формы транзакции
- **WHEN** пользователь создаёт подтег, выбрав родительский тег в форме
- **THEN** подтег сохраняется и доступен в селекторе для текущей и будущих транзакций

### Requirement: Отображение иерархии

Дерево тегов SHALL отображаться как вложенный список или дерево с визуальной иерархией.

#### Scenario: Отображение пути тега
- **WHEN** пользователь выбирает подтег в форме или таблице
- **THEN** отображается полный путь, например «Еда / Кафе»
