### Requirement: Опциональная аутентификация v1

В v1 аутентификация MAY быть реализована как HTTP Basic Auth или простой логин/пароль. Приложение предназначено для личного использования одним пользователем.

#### Scenario: Доступ без auth в dev
- **WHEN** auth отключён в конфигурации (режим разработки)
- **THEN** API и UI доступны без учётных данных

#### Scenario: HTTP Basic Auth при деплое
- **WHEN** auth включён и пользователь запрашивает защищённый ресурс без credentials
- **THEN** сервер возвращает 401 с заголовком `WWW-Authenticate: Basic`

#### Scenario: Логин по паролю
- **WHEN** включён режим логин/пароль и пользователь вводит верные credentials
- **THEN** выдаётся сессия или токен, последующие запросы проходят авторизацию

### Requirement: HTTPS при деплое

Продакшен-деплой SHALL использовать HTTPS.

#### Scenario: Редирект HTTP → HTTPS
- **WHEN** клиент обращается по HTTP в production
- **THEN** запрос перенаправляется на HTTPS (или отклоняется)
