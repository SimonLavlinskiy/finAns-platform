# Импорт транзакций из CSV

## Зачем

Пользователь ведёт часть операций во внешних системах (выгрузки банка, таблицы) и сейчас обязан переносить каждую запись в форму транзакции вручную. Нужен массовый импорт из CSV с проверкой и модерацией перед сохранением, чтобы перенос истории операций не занимал часы ручного ввода.

## Что меняется

- Новая страница `/import` с drag-and-drop загрузкой `.csv`; файл отправляется на backend, парсинг и валидация выполняются на сервере.
- Иконка-вопрос рядом с кнопкой «Выбрать файл», открывающая справку по формату файла (обязательные/необязательные колонки, формат каждого поля, пример строки).
- Новая таблица БД `moderation_transactions` (плюс `import_batches` для группировки строк одной загрузки) — распарсенные строки сохраняются туда, а не сразу в `transactions`.
- Валидация каждой строки на сервере: обязательные поля (`title`, `amount`, `date`), формат суммы, формат даты, допустимые значения `category`, существование `tag` в системе (с поддержкой иерархии через `/`).
- Таблица модерации (фронт читает/правит строки через API) с inline-редактированием всех полей, статусами строки на сервере (`Ожидает` / `Готово` / `Ошибка`) и автоматическим пересчётом статуса после правок. «Перенесено» — не серверный статус: строка удаляется из таблицы модерации в момент переноса в `transactions`, на фронте это лишь кратковременная анимация перед исчезновением строки из списка.
- Массовое и одиночное принятие готовых строк → запись **перемещается** из `moderation_transactions` в `transactions` (создаётся транзакция, строка модерации помечается перенесённой/удаляется).
- Завершающий баннер импорта со сводкой и переходом в `/transactions`.

## Non-цели

- Импорт форматов кроме CSV (XLSX, OFX и т.п.) — вне scope.
- Создание новых тегов массово из файла — тег должен существовать в системе до импорта (строка без найденного тега получает статус `Ошибка`, а не создаёт тег автоматически).
- Хранение исходного файла CSV целиком (сохраняются только распарсенные строки в `moderation_transactions`, не сам файл).

## Capabilities

### Новые capabilities
- `import-transactions`: загрузка CSV на backend, серверный парсинг и валидация, хранение строк в `moderation_transactions` до согласования, таблица модерации с inline-редактированием через API, массовое/одиночное принятие строк (перемещение в `transactions`), итоговая сводка импорта.

## Влияние

- `workspace/finAns-backend/`: новые таблицы `import_batches` и `moderation_transactions` (миграция), сервис парсинга/валидации CSV, эндпоинты загрузки/чтения/правки/принятия строк, логика переноса строки в `transactions` при принятии.
- `workspace/finAns-frontend/`: новая страница `/import`, загрузка файла на backend, таблица модерации на данных из API (переиспользует `TagFormPicker` из формы транзакции для поля «Тег»), inline-редактирование через PATCH, принятие строк через API.
- Миграция БД обязательна — добавляются `import_batches` и `moderation_transactions`.
