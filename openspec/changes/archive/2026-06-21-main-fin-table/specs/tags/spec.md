## ADDED Requirements

### Requirement: Цвет тега

Каждый тег SHALL иметь цвет (`color`, hex) из пресетного набора (~10 цветов); цвет отображается в pill-бейдже в таблице и в дереве тегов.

#### Scenario: Создание тега с цветом
- **WHEN** пользователь создаёт тег и выбирает цвет `#22C55E`
- **THEN** тег сохраняется с полем `color` и отображается с этим цветом

#### Scenario: Цвет в breadcrumb таблицы
- **WHEN** транзакция привязана к подтегу с цветом
- **THEN** pill-бейдж тега в таблице использует цвет тега

### Requirement: Страница управления тегами `/tags`

Приложение SHALL отображать дерево тегов с действиями: `+ Подтег`, редактирование, удаление; кнопка `+ Добавить тег` в корне.

#### Scenario: Инлайн-создание в дереве
- **WHEN** пользователь нажимает `+ Добавить тег` на странице `/tags`
- **THEN** появляется инлайн-форма (Название, Цвет, Родитель опционально) без перехода на другую страницу

#### Scenario: Удаление тега с подтегами
- **WHEN** пользователь удаляет тег, у которого есть `children`
- **THEN** система предупреждает об удалении всего поддерева и запрашивает подтверждение

#### Scenario: Удаление используемого тега
- **WHEN** тег используется в N транзакциях
- **THEN** предупреждение: «Тег используется в N записях…»; после подтверждения тег снимается с записей

### Requirement: Inline-создание тега в форме транзакции

В tree select формы транзакции SHALL быть опция `+ Создать тег "..."` с мини-формой (название, родитель, цвет) прямо в дропдауне.

#### Scenario: Создание и автовыбор
- **WHEN** пользователь создаёт тег из формы транзакции
- **THEN** новый тег сразу выбирается в поле «Тег» текущей формы

### Requirement: REST API тегов

Backend SHALL предоставлять: `GET /api/v1/tags` (вложенное дерево), `POST /api/v1/tags`, `PUT /api/v1/tags/:id`, `DELETE /api/v1/tags/:id`, `GET /api/v1/tags/:id/usage`.

#### Scenario: Дерево тегов
- **WHEN** клиент запрашивает `GET /api/v1/tags`
- **THEN** ответ — массив корневых тегов с вложенными `children`

#### Scenario: Usage count
- **WHEN** клиент запрашивает `GET /api/v1/tags/7/usage`
- **THEN** ответ `{ "count": N }` — количество транзакций с этим тегом
