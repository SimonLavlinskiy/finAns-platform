# Спецификация: mandatory-payments

Capability: обязательные регулярные платежи с периодичностью и автосдвигом даты.

---

## ADDED

### Requirement: Список обязательных платежей

Система SHALL отображать все обязательные платежи пользователя в виде таблицы со столбцами: Наименование, Сумма, Категория (тег), Периодичность, Дата платежа, Чекбокс оплаты, Меню действий.

#### Scenario: Пустой список

GIVEN нет ни одного обязательного платежа
WHEN пользователь открывает раздел «План»
THEN отображается сообщение «Нет платежей» и кнопка «Добавить первый платёж»

#### Scenario: Список с платежами

GIVEN есть один или несколько обязательных платежей
WHEN пользователь открывает раздел «План»
THEN таблица показывает все платежи с корректными значениями во всех столбцах

---

### Requirement: Подсветка приближающейся даты платежа

Система SHALL подсвечивать ячейку «Дата платежа» тёплым жёлтым цветом, если до даты платежа осталось 3 дня или менее и дата ещё не наступила.

#### Scenario: Платёж через 3 дня

GIVEN платёж с `next_payment_date` = сегодня + 3 дня
WHEN пользователь смотрит на таблицу
THEN ячейка даты окрашена тёплым жёлтым цветом

#### Scenario: Платёж через 4 дня

GIVEN платёж с `next_payment_date` = сегодня + 4 дня
WHEN пользователь смотрит на таблицу
THEN ячейка даты отображается обычным цветом

#### Scenario: Дата уже прошла

GIVEN платёж с `next_payment_date` = вчера
WHEN пользователь смотрит на таблицу
THEN ячейка даты отображается обычным цветом (без жёлтой подсветки)

---

### Requirement: Отметка факта оплаты (чекбокс)

Система SHALL при установке чекбокса «оплачено» автоматически пересчитать `next_payment_date` = старая дата + интервал периодичности, после чего чекбокс сбрасывается в непроставленное состояние.

#### Scenario: Отметка ежемесячного платежа

GIVEN платёж с `recurrence = monthly` и `next_payment_date = 2026-06-15`
WHEN пользователь ставит галочку
THEN `next_payment_date` становится `2026-07-15`, галочка снята, цвет ячейки пересчитан

#### Scenario: Отметка еженедельного платежа

GIVEN платёж с `recurrence = weekly` и `next_payment_date = 2026-06-22`
WHEN пользователь ставит галочку
THEN `next_payment_date` становится `2026-06-29`, галочка снята

#### Scenario: Отметка ежеквартального платежа

GIVEN платёж с `recurrence = quarterly` и `next_payment_date = 2026-04-01`
WHEN пользователь ставит галочку
THEN `next_payment_date` становится `2026-07-01`, галочка снята

#### Scenario: Отметка полугодового платежа

GIVEN платёж с `recurrence = semi_annual` и `next_payment_date = 2026-01-10`
WHEN пользователь ставит галочку
THEN `next_payment_date` становится `2026-07-10`, галочка снята

### Requirement: Создание обязательного платежа

Система SHALL позволять создать обязательный платёж через боковую панель (Sheet) с полями: Наименование, Сумма, Категория (тег), Периодичность, Дата платежа. Все поля обязательны.

#### Scenario: Успешное создание

GIVEN пользователь заполнил все поля
WHEN нажимает «Сохранить»
THEN платёж создан, строка появляется в таблице, Sheet закрывается

#### Scenario: Создание с незаполненным полем

GIVEN пользователь не выбрал тег
WHEN нажимает «Сохранить»
THEN Sheet остаётся открытым, под пустым полем отображается ошибка «обязательно»

#### Scenario: Кнопка «Сохранить и ещё»

GIVEN пользователь заполнил все поля
WHEN нажимает «Сохранить и ещё»
THEN платёж создан, форма сбрасывается (Sheet остаётся открытым)

---

### Requirement: Редактирование обязательного платежа

Система SHALL позволять редактировать любой платёж через меню «…» → «Редактировать», открывая тот же Sheet с заполненными полями.

#### Scenario: Успешное редактирование

GIVEN пользователь изменил сумму платежа
WHEN нажимает «Сохранить»
THEN платёж обновлён, таблица обновляется, Sheet закрывается

---

### Requirement: Дублирование обязательного платежа

Система SHALL создавать копию платежа с теми же параметрами через меню «…» → «Дублировать». Дата платежа у копии = сегодня.

#### Scenario: Дублирование

GIVEN пользователь выбрал «Дублировать» из меню «…»
WHEN операция успешна
THEN в таблице появляется новая строка с теми же значениями, `next_payment_date = today`

---

### Requirement: Удаление обязательного платежа

Система SHALL удалять платёж через меню «…» → «Удалить» с подтверждением.

#### Scenario: Удаление с подтверждением

GIVEN пользователь выбрал «Удалить»
WHEN подтверждает в диалоге
THEN платёж удалён, строка исчезает из таблицы

#### Scenario: Отмена удаления

GIVEN пользователь выбрал «Удалить»
WHEN отменяет в диалоге
THEN платёж остаётся в таблице

---

### Requirement: Варианты периодичности

Система SHALL поддерживать следующие значения `recurrence`: `daily` (Ежедневно), `weekly` (Еженедельно), `monthly` (Ежемесячно), `quarterly` (Ежеквартально), `semi_annual` (Раз в полгода), `yearly` (Ежегодно).

#### Scenario: Все варианты доступны в форме

GIVEN пользователь открывает Sheet создания платежа
WHEN кликает на поле «Периодичность»
THEN в выпадающем списке 6 вариантов: Ежедневно, Еженедельно, Ежемесячно, Ежеквартально, Раз в полгода, Ежегодно
