## ADDED Requirements

### Requirement: Создание учётной записи
Система SHALL позволять создать пользователя через административную форму в UI: поля `display_name` (отображаемое имя) и `username` (уникальный логин). `username` — латиница, цифры, `_`, длина 3–20 символов, уникален без учёта регистра.

#### Scenario: Успешное создание пользователя
- **WHEN** администратор отправляет форму с `display_name="Simon"` и `username="simon"`
- **THEN** создаётся запись `users(id, username="simon", display_name="Simon", created_at)` и форма закрывается

#### Scenario: Дубликат username
- **WHEN** создаётся пользователь с `username`, который уже существует (без учёта регистра)
- **THEN** возвращается ошибка `USERNAME_TAKEN` и запись не создаётся

#### Scenario: Невалидный username
- **WHEN** `username` содержит пробелы, спецсимволы или длиннее 20 / короче 3 символов
- **THEN** возвращается ошибка валидации `INVALID_USERNAME`

### Requirement: Вход без пароля
Система SHALL предоставлять экран входа: список существующих пользователей и поле ввода `@username`. После выбора/ввода активный пользователь сохраняется в `localStorage` и используется во всех последующих запросах.

#### Scenario: Вход существующего пользователя через список
- **WHEN** пользователь кликает на своё имя в списке на экране входа
- **THEN** `localStorage.activeUserId` устанавливается в `id` пользователя и происходит переход на экран выбора проекта

#### Scenario: Вход по @username
- **WHEN** пользователь вводит `@simon` в поле и подтверждает
- **THEN** если `username="simon"` существует — вход выполнен; иначе ошибка `USER_NOT_FOUND`

#### Scenario: Повторный визит с сохранённым пользователем
- **WHEN** `localStorage.activeUserId` установлен при загрузке приложения
- **THEN** экран входа пропускается, пользователь переходит сразу к выбору/переключению проекта

### Requirement: Список пользователей
Система SHALL предоставлять API `GET /api/v1/users` — список всех пользователей (`id`, `username`, `display_name`). Используется для экрана входа и поиска при приглашении в проект.

#### Scenario: Получение списка
- **WHEN** клиент запрашивает `GET /api/v1/users`
- **THEN** возвращается массив пользователей; если пользователей нет — пустой массив

#### Scenario: Поиск по username
- **WHEN** клиент запрашивает `GET /api/v1/users?username=simon`
- **THEN** возвращается массив с пользователями, чей `username` соответствует запросу (точное совпадение, case-insensitive)
