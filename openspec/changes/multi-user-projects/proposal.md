# Многопользовательский режим: пользователи и проекты

## Зачем

Сейчас все данные принадлежат единому набору без изоляции — нельзя вести несколько независимых бюджетов или приглашать соавторов. Вводятся понятия пользователя и проекта, чтобы несколько людей могли совместно вести учёт финансов в изолированных контейнерах данных.

## Что меняется

- **BREAKING**: Вводится сущность `project` — все существующие данные (транзакции, теги, лимиты, обязательные платежи, плановые расходы, баланс) привязываются к `project_id`
- **BREAKING**: Вводится сущность `user` с уникальным `@username`; вход без пароля — выбор пользователя из списка
- **BREAKING**: Все API-эндпоинты получают контекст проекта (через заголовок `X-Project-ID` или path-сегмент)
- Миграция данных: создаётся дефолтный пользователь `@simon` + дефолтный проект, все существующие записи привязываются к нему; два существующих пользователя подвязываются к проекту
- Переключатель активного проекта в хедере (если пользователь состоит в нескольких)
- Раздел «Участники» в настройках проекта: добавление по `@username`, роли `owner` / `member`
- Создание пользователей — через административную форму в UI (не seed-скрипт)

## Non-цели

- Полноценная аутентификация (пароль, email, OAuth) — v2
- Push/email-уведомления о приглашении в проект
- Тонкая настройка прав (сейчас только `owner` / `member`)
- Передача владения проектом
- История изменений / audit log
- Подтверждение приглашения приглашённым: участник добавляется мгновенно (без подтверждения)

## Capabilities

### Новые capabilities

- `user-accounts`: Создание и управление учётными записями (`@username`, без пароля); вход выбором пользователя из списка; создаёт `specs/user-accounts/spec.md`
- `projects`: Создание проекта, переключение между проектами, приглашение участников по `@username`, роли `owner` / `member`; создаёт `specs/projects/spec.md`

### Изменяемые capabilities

- `transactions`: Добавляется `project_id` — все запросы фильтруются по активному проекту
- `tags`: Дерево тегов становится per-project
- `spending-limits`: Лимиты становятся per-project
- `mandatory-payments`: Обязательные платежи становятся per-project
- `planned-expenses`: Плановые расходы и категории становятся per-project
- `balance`: Баланс и начальная сумма становятся per-project

## Влияние

- `workspace/finAns-backend/`: новые таблицы `users`, `projects`, `project_members`; миграция добавляет `project_id` во все таблицы данных; новые хендлеры для пользователей и проектов; middleware извлечения активного проекта из заголовка
- `workspace/finAns-frontend/`: экран выбора пользователя (вход), экран создания проекта, переключатель проекта в хедере, раздел «Участники» в настройках; передача `X-Project-ID` во все API-запросы; хранение выбранного проекта в `localStorage`
- Миграции: `000010_add_users_and_projects.up.sql` (новые таблицы + `project_id` во все существующие), data-migration seed для существующих данных
