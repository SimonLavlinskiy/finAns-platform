# Доска планируемых расходов (карточки категорий)

## Зачем

Раздел «Расходы» сейчас не реализован (placeholder на фронтенде), а существующая спека `planned-expenses` описывает простую таблицу без категорий, приоритетов и архива. Нужен полноценный визуальный инструмент планирования покупок: карточки категорий со своим цветом, приоритеты с авто-подсветкой горящих сроков, архив выполненных покупок и быстрый доступ к товару по ссылке — вместо плоского списка и прямой конвертации в транзакцию.

## Что меняется

- Полностью переработана модель планируемого расхода: вместо тега и прямой конвертации в транзакцию — собственная категория (название + цвет), приоритет (низкий/средний/высокий), статус (активен/в архиве). **BREAKING** для существующей спеки `planned-expenses` (поле `тег` и действие «Исполнено → транзакция» удаляются).
- Новая сущность «категория планируемых расходов»: название, цвет (свотч из палитры), порядок отображения карточки. Создаётся либо явно, либо инлайн прямо из формы товара.
- Боковая панель (Sheet) создания/редактирования товара: Наименование, Стоимость, Дата, Ссылка, Категория (выбор существующей или создание новой с цветом), Приоритет. Кнопки «Сохранить» и «Сохранить и ещё».
- Сетка цветных карточек-категорий (2–3 в строке, 1 на узких экранах, равная высота в строке), с подсказкой для пустой категории.
- Список товаров внутри карточки: точка приоритета, опциональная стоимость/дата, кликабельная ссылка (отдельный цвет текста), сортировка по приоритету (убывание) и дате добавления внутри группы.
- Авто-эскалация приоритета до «Высокий» и тёплая жёлтая подсветка даты, если до срока ≤3 дней (и срок не прошёл) — независимо от вручную заданного приоритета.
- Действие «…»/карандаш на товаре (виден только при hover): Выполнено (→ архив, без удаления данных), Редактировать, Удалить.
- Архив выполненных товаров с сохранением всех полей.
- Drag-and-drop переупорядочивание карточек категорий с сохранением порядка.
- Автоматический подбор цвета текста (тёмный/светлый) на цветной карточке по контрасту с её фоном.

## Non-цели

- Многопользовательский доступ / права — приложение однопользовательское (как и весь v1).
- Уведомления о горящих сроках (push/email) — только визуальная подсветка в UI.
- Бюджетирование по категориям (лимиты трат на категорию) — пересекается с будущей capability `spending-limits`, не входит в этот change.
- Перенос архивных товаров обратно в активный список — не описан в исходном функциональном ТЗ, остаётся возможной задачей на будущее.
- Реюз дерева тегов (`tags`) для категорий — категории планируемых расходов сознательно отделены от тегов транзакций (см. design.md), поэтому изменений в capability `tags` не требуется.

### Изменяемые capabilities
- `planned-expenses`: полная переработка требований — категории со своим цветом вместо тега, приоритет с авто-эскалацией по сроку, действие «Выполнено» переносит товар в архив (а не создаёт транзакцию), карточный UI с drag-and-drop порядком категорий.

## Влияние

- `workspace/finAns-backend/`: новая миграция (категории планируемых расходов + переработка таблицы `planned_expenses` под приоритет/статус/архив/порядок категорий), новый domain/service/repository/handler слой (`PlannedExpense`, `PlannedExpenseCategory`), REST-эндпоинты CRUD для категорий и товаров + `POST /planned-expenses/{id}/complete` (архивация) + `PATCH /planned-expense-categories/reorder`.
- `workspace/finAns-frontend/`: замена placeholder `PlannedExpensesPage.tsx` на карточную доску, новый Sheet-компонент формы товара с инлайн-созданием категории и цветовым свотч-пикером, добавление библиотеки drag-and-drop (на фронтенде сейчас её нет), утилита авто-контраста текста на цветном фоне, логика авто-эскалации приоритета по дате.
- API: новые DTO/контракты для категорий и товаров планируемых расходов; старые поля `tag_id`/`category`/`specificity` для этой сущности больше не используются.
